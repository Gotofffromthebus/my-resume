# GSAP Best Practices & Instructions

This project follows the official **GSAP AI Skills** for high-performance and maintainable animations.

## 🚀 Core Principles

- **Performance First:** Use `transform` aliases (`x`, `y`, `scale`, `rotation`, `xPercent`, `yPercent`) instead of layout properties (`top`, `left`, `width`, `height`).
- **Use Timelines:** Prefer `gsap.timeline()` for sequencing multiple animations instead of using `delay`.
- **Proper Cleanup:** Always ensure animations are killed or reverted when components unmount or elements are removed.
- **CamelCase:** Use camelCase for all property names (e.g., `backgroundColor`, `borderRadius`).
- **Visibility:** Use `autoAlpha` instead of `opacity` to toggle `visibility: hidden` and prevent interaction with invisible elements.

## ⚛️ React Integration (if applicable)

- **Use `useGSAP()`:** Always use the `@gsap/react` hook instead of `useEffect`.
- **Scoping:** Use a `ref` for the container and pass it as the `scope` to `useGSAP()`. This allows using simple selector strings safely.
- **Cleanup:** `useGSAP()` handles cleanup automatically.
- **ContextSafe:** Wrap animations in event handlers (like clicks) with `contextSafe` from `useGSAP()`.

## ⚡ Performance Optimization

- **`gsap.quickTo()`:** Use for properties updated many times per second (e.g., mouse followers).
- **Layer Promotion:** Use `will-change: transform` in CSS for frequently animated elements.
- **Staggers:** Use the built-in `stagger` property instead of manual loops for animating multiple elements.
- **Batching:** Group DOM reads and writes; GSAP handles writes efficiently.
